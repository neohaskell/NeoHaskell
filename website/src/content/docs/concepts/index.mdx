The tutorial showed you *how* to do things in NeoHaskell. This section explains *why*. If the tutorial was about moves, this is about strategy. You don't need these pages to be productive — you can build real systems using only the tutorial patterns — but reading them will change how you think about state, consistency, and domain modeling.

## Why it's in this order

The early pages bridge you from the mental model you probably already have (CRUD) to the mental model NeoHaskell assumes (events). The middle pages break down the specific primitives — commands, queries, services. The later pages cover the harder questions you'll run into once you've shipped something.

You can jump around, but the earlier pages often refer back to each other, and the later pages assume you've read the earlier ones.
