## Before You Start

{/*
  Prerequisites: minimal. Just a terminal and internet.
  Explicitly state "you don't need" to address common assumptions.
*/}

- What you need: a terminal and an internet connection

:::note[You don't need any of this]
- Docker or Kubernetes (Misconception #16)
- Haskell installed separately
- GHC, Cabal, or Stack
- Any prior Nix experience
- NeoHaskell uses **Nix** to manage its toolchain — but you don't need to know Nix. The installer handles everything.
:::

## Troubleshooting

{/*
  DOCS-PLAN says: "The 5 most common installation failures and their fixes, based on Discord reports"
  These should be real failure modes, not hypothetical ones.
*/}

### Top 5 Installation Issues

1. **"command not found: nix"** — The Nix installer needs a new shell session. Close your terminal and open a new one.
2. **"error: experimental features 'nix-command flakes' are disabled"** — NeoHaskell uses Nix Flakes. The installer should enable this automatically. If not: show the manual config line.
3. **macOS Gatekeeper blocks the download** — System Preferences → Security → Allow. Show the exact click path.
4. **"disk space"** — Nix needs ~2GB free. If you're low on space, the error message tells you.
5. **WSL-specific: "unable to access the network"** — Windows firewall/proxy settings. Link to the WSL networking troubleshooting guide.
