You are five minutes away from a working NeoHaskell environment. The whole installation is three steps:

1. Install [Determinate Nix](https://determinate.systems/nix).
2. Add yourself to Nix's trusted users list.
3. Clone the NeoHaskell template and run `nix develop` inside it.

That's the whole thing. No compiler toolchain to install by hand. No version juggling. Nix takes care of the compiler, the standard library, the editor integration, and the build tooling. You do not need to know anything about Nix to use it — treat it as a black box that gives you a working shell.

## What you'll have at the end

- A shell where `cabal build`, `cabal run`, `cabal test`, and your editor's language server all work.
- The NeoHaskell template project on your disk, ready to modify.
- A fully-specified environment pinned to a known-good set of tool versions.

## What you don't need

<Aside type="note" title="Things people assume they need — and don't">
- **Docker or Kubernetes.** Not now, not for deployment, not ever unless you want to.
- **A compiler installed separately.** Nix provides it.
- **Any build tool configured by hand.** Nix provides those too.
- **Prior Nix experience.** You will run exactly three Nix commands in this guide and none afterwards.
- **An existing project to adapt.** The template is self-contained.
</Aside>

## Supported platforms

NeoHaskell runs on:

- **macOS** (Apple Silicon and Intel)
- **Linux** (x86_64 and ARM64)
- **Windows via WSL2** (install Determinate Nix inside WSL, not on Windows directly)

If you are on Windows without WSL, [follow Microsoft's WSL2 setup guide](https://learn.microsoft.com/en-us/windows/wsl/install) first, then come back.

## Step 1 — Install Determinate Nix

Determinate Nix is a polished installer for Nix with sensible defaults. It is the fastest way to get a working Nix installation.

<Tabs syncKey="platform">
  <TabItem label="macOS / Linux / WSL">
    Run this command in your terminal:

    ```bash
    curl -fsSL https://install.determinate.systems/nix | sh -s -- install
    ```

    The installer will ask you a few questions. Accept the defaults unless you know you want something different. When it finishes, close your terminal and open a new one — this refreshes your `PATH` so the `nix` command is available.

    Verify the installation:

    ```bash
    nix --version
    ```

    You should see a line like `nix (Determinate Nix 3.x.x)`.
  </TabItem>
</Tabs>

<Aside type="tip" title="Already have Nix?">
If you already have Nix installed from another source (nix.org, Lix, etc.), you do not need to reinstall. Just make sure **flakes are enabled** in your `nix.conf`:

```conf
experimental-features = nix-command flakes
```

Determinate Nix has flakes on by default, which is why we recommend it.
</Aside>

## Step 2 — Add yourself as a trusted user

Nix runs build workers in a sandbox. To let the NeoHaskell template use its shared binary cache (which makes the first `nix develop` take about two minutes instead of thirty), you need to be on the trusted users list.

<Steps>
1. **Edit the Nix daemon config file.** Its location depends on your install:

   ```bash
   sudo nano /etc/nix/nix.conf
   ```

2. **Add a line** with your username (replace `your-username` with the output of `whoami`):

   ```conf
   trusted-users = root your-username
   ```

   If the line already exists, append your username after the existing ones, separated by a space.

3. **Restart the Nix daemon.** On macOS:

   ```bash
   sudo launchctl kickstart -k system/org.nixos.nix-daemon
   ```

   On Linux with systemd:

   ```bash
   sudo systemctl restart nix-daemon
   ```
</Steps>

You only need to do this once per machine.

## Step 3 — Clone the template and enter the dev shell

<Steps>
1. **Clone the NeoHaskell starter template** to a directory of your choice:

   ```bash
   git clone https://github.com/neohaskell/template my-first-app
   cd my-first-app
   ```

2. **Enter the development shell:**

   ```bash
   nix develop
   ```

   The first time you run this, Nix downloads GHC, the NeoHaskell core library, and all build tooling. This takes a couple of minutes on a decent internet connection. The output will look busy — a lot of files scroll by, and Nix asks to use the binary cache (say yes).

   When it finishes, your shell prompt will change to indicate you are inside the dev shell. You now have `cabal`, `ghc`, `hlint`, `fourmolu`, and every other tool the project uses.

3. **Verify everything works:**

   ```bash
   cabal build
   ```

   The first build compiles the example application. On modern hardware this takes about thirty seconds. Subsequent builds are incremental and much faster.

4. **Run the example:**

   ```bash
   cabal run
   ```

   You should see the example app start up. If it does, you're done — your environment is ready.
</Steps>

## What the template gives you

The template is a minimal, runnable application structured the way you'll structure real projects:

```text
my-first-app/
├── flake.nix              # Nix environment (don't modify unless you know why)
├── cabal.project          # Cabal build config
├── my-app.cabal           # Package definition
├── src/
│   ├── App.hs             # Application entry point
│   └── MyApp/
│       ├── Config.hs      # Typed configuration
│       └── Domain/        # Your commands, events, entities, queries
├── tests/
│   └── Spec.hs            # Test entry point
└── uploads/               # File upload directory (if using file uploads)
```

You will spend nearly all your time in `src/MyApp/Domain/`. The other files you'll modify once during setup and then rarely touch.

## Editor setup

NeoHaskell works with any editor that supports its language server (HLS, which the template ships with).

<Tabs syncKey="editor">
  <TabItem label="VS Code">
    Install the language server extension for NeoHaskell from the VS Code marketplace (search for "HLS"). When you open the project inside the `nix develop` shell, the extension will pick up the language server automatically.

    To make VS Code always use the dev shell, install [`direnv`](https://direnv.net/) and the VS Code **direnv** extension, then run:

    ```bash
    echo "use flake" > .envrc
    direnv allow
    ```

    Now opening the project folder in VS Code automatically loads the dev shell — no `nix develop` needed.
  </TabItem>
  <TabItem label="Neovim">
    Any LSP-capable config works. The easiest path is [nvim-lspconfig](https://github.com/neovim/nvim-lspconfig) configured to use HLS as the LSP server. Run Neovim from inside the `nix develop` shell so it finds the server on your `PATH`.

    Or, as with VS Code, use [`direnv`](https://direnv.net/) and launch Neovim from the project directory — direnv loads the shell transparently.
  </TabItem>
  <TabItem label="Emacs">
    Use [`lsp-mode`](https://emacs-lsp.github.io/lsp-mode/) or [`eglot`](https://github.com/joaotavora/eglot) pointed at HLS. Launch Emacs from the dev shell, or use [`envrc-mode`](https://github.com/purcell/envrc) to load the shell automatically on project visits.
  </TabItem>
  <TabItem label="Other">
    Any editor with LSP support works. Configure it to run HLS as the LSP server for `.hs` files, and make sure the editor launches from inside the `nix develop` shell (or uses direnv).
  </TabItem>
</Tabs>

## Daily workflow

Once installed, your daily loop looks like this:

```bash
cd my-first-app
nix develop           # Enter the dev shell (skipped automatically if you use direnv)
cabal build           # Build
cabal test            # Run tests
cabal run             # Run the example
```

If you forget `nix develop` and run `cabal build` outside the shell, it will fail because `cabal` is not on your `PATH` — that's expected. The dev shell is what makes everything work.

## Troubleshooting

### "command not found: nix" after installation

Close your terminal and open a new one. The installer adds Nix to your shell's init scripts, which only take effect on new sessions.

### "experimental features 'nix-command flakes' are disabled"

You installed Nix from a source other than Determinate, and flakes are off by default. Edit `~/.config/nix/nix.conf` (creating it if needed) and add:

```conf
experimental-features = nix-command flakes
```

### `nix develop` takes forever

The first run downloads a few hundred megabytes of toolchain. If it feels stuck but your network traffic is still flowing, it is working — just slow. If you haven't completed Step 2 (trusted users), you'll miss the binary cache and have to build GHC from source, which takes much longer. Double-check the trusted-users line before retrying.

### Editor does not find the language server

Your editor is not launching from inside the `nix develop` shell. Either launch your editor from a terminal that already ran `nix develop`, or set up `direnv` as described in the editor section.

### Windows without WSL

NeoHaskell does not support native Windows directly. Install WSL2, run Ubuntu or Debian inside it, and follow the Linux instructions.
