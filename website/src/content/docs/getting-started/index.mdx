## Which order should I read them in?

**If you are impatient and want to write code now:**

1. Installation
2. Your first events
3. Cheat sheet (keep it open)
4. Jump into [the NeoBank tutorial](/tutorial/)

**If you prefer to understand the syntax before touching it:**

1. Installation
2. Reading NeoHaskell
3. Syntax warm-up
4. Your first events
5. [NeoBank tutorial](/tutorial/)

**If you are using an AI assistant:**

1. Installation
2. Using AI (paste the system prompt into your assistant)
3. Your first events (with the assistant as your pair)
4. [NeoBank tutorial](/tutorial/)

Both paths end in the same place: you know the basics, and you're ready for the tutorial.

## What you'll know by the end

- How to set up a NeoHaskell project from scratch.
- How to read any NeoHaskell file without prior Haskell experience.
- The core moves: records, pattern matching, pipes, `do` blocks, `Maybe`, `Result`, `Task`.
- How to write your first command, entity, and event.
- How to use AI assistants productively.

When you finish this section, your next stop is [**Build NeoBank**](/tutorial/) — the full tutorial that teaches you the language by building a real banking service.
