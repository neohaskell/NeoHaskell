You've read about the syntax. Now let's build muscle memory. Work through these ten exercises in order. Each takes about thirty seconds. By the end, the language will feel familiar, not alien.

You can do these in any NeoHaskell module — create a scratch file inside your template project, or use `ghci` from the dev shell if you prefer a REPL.

## Exercise 1 — Define a record

**Goal:** Make a record type for a book.

```haskell
data Book = Book
  { title :: Text
  , author :: Text
  , pages :: Int
  }
  deriving (Show)
```

**Test it:** Construct one.

```haskell
myBook :: Book
myBook = Book
  { title = "The Goal"
  , author = "Eliyahu Goldratt"
  , pages = 384
  }
```

**Key takeaway:** Commas go at the start of each line, not the end. Field names are explicit. `deriving (Show)` lets you print the value for debugging.

---

## Exercise 2 — Read a field

**Goal:** Get the title of `myBook`.

```haskell
title :: Text
title = myBook.title
```

**Key takeaway:** Dot access, just like JavaScript or Python.

---

## Exercise 3 — Update a field

**Goal:** Make a copy of `myBook` with one more page.

```haskell
longerBook :: Book
longerBook = myBook { pages = myBook.pages + 1 }
```

**Key takeaway:** Record update creates a new record — it never mutates the original. `myBook` still has its original `pages` value.

---

## Exercise 4 — Pattern match with `case`

**Goal:** Categorize a book by its page count.

```haskell
sizeLabel :: Book -> Text
sizeLabel book = case book.pages of
  n | n < 100 -> "pamphlet"
  n | n < 300 -> "short"
  n | n < 600 -> "average"
  _ -> "tome"
```

**Key takeaway:** `case` with guards (`|`) is how you express "if this, elif that, else." The underscore `_` catches anything not matched above.

---

## Exercise 5 — Pipe a value through functions

**Goal:** Take a book title, uppercase it, and trim whitespace.

```haskell
import Text qualified

cleanTitle :: Book -> Text
cleanTitle book =
  book.title
    |> Text.trim
    |> Text.toUpper
```

**Key takeaway:** `|>` reads left-to-right. `book.title` is the value, `Text.trim` transforms it, `Text.toUpper` transforms the result of that. You can read this as "take the title, trim it, then uppercase it."

---

## Exercise 6 — Work with `Maybe`

**Goal:** Write a function that returns the first character of a title, or `Nothing` if the title is empty.

```haskell
import Maybe qualified
import Text qualified

firstLetter :: Text -> Maybe Char
firstLetter text = case Text.uncons text of
  Nothing -> Nothing
  Just (c, _) -> Just c
```

**Test it:**

```haskell
firstLetter "NeoHaskell"  -- Just 'N'
firstLetter ""            -- Nothing
```

**Key takeaway:** `Maybe` forces you to think about "what if it's missing?" The compiler will not let you skip the `Nothing` case.

---

## Exercise 7 — Work with `Result`

**Goal:** Parse an integer from text, returning either `Ok n` or `Err reason`.

```haskell
import Int qualified

parseAge :: Text -> Result Text Int
parseAge text = case Int.fromText text of
  Nothing -> Err "not a valid integer"
  Just n -> if n < 0
    then Err "age cannot be negative"
    else Ok n
```

**Test it:**

```haskell
parseAge "42"    -- Ok 42
parseAge "-3"    -- Err "age cannot be negative"
parseAge "hello" -- Err "not a valid integer"
```

**Key takeaway:** `Result err ok` expresses "this might fail, and here's why." Unlike exceptions, failures are part of the type — you have to handle them.

---

## Exercise 8 — A sum type

**Goal:** Model a traffic light with three colors.

```haskell
data Light
  = Red
  | Yellow
  | Green
  deriving (Eq, Show)

nextLight :: Light -> Light
nextLight light = case light of
  Red -> Green
  Green -> Yellow
  Yellow -> Red
```

**Key takeaway:** This is the most basic sum type — three variants with no fields. The compiler will reject `nextLight` if you miss a case. Try commenting out one branch and watch it scream.

---

## Exercise 9 — A sum type with fields

**Goal:** Model a payment event that can be a deposit or a withdrawal, each with an amount.

```haskell
data PaymentEvent
  = Deposited { amount :: Int }
  | Withdrawn { amount :: Int }
  deriving (Show)

describe :: PaymentEvent -> Text
describe event = case event of
  Deposited { amount } -> [fmt|Deposited $#{amount}|]
  Withdrawn { amount } -> [fmt|Withdrew $#{amount}|]
```

**Test it:**

```haskell
describe (Deposited { amount = 50 })  -- "Deposited $50"
describe (Withdrawn { amount = 20 })  -- "Withdrew $20"
```

**Key takeaway:** Sum types with record fields are how you describe "one of several different shapes of data." Every event type in NeoHaskell looks like this. Fold over them with `case`.

---

## Exercise 10 — A `do` block

**Goal:** Write a task that reads a name from the console and prints a greeting.

```haskell
import Console qualified

greet :: Task Text Unit
greet = do
  Console.print "What is your name?"
  name <- Console.readLine
  Console.print [fmt|Hello, #{name}!|]
```

**Key takeaway:** `do` blocks sequence steps. `<-` captures a value from a step that returns something. Steps without `<-` are run for their effect. This is async/await with a different name.

---

## Quick reference — the moves you just learned

| Move | Syntax |
|---|---|
| Define a record | `data Foo = Foo { a :: Int, b :: Text }` |
| Construct a record | `Foo { a = 1, b = "x" }` |
| Read a field | `foo.a` |
| Update a field | `foo { a = 42 }` |
| Pattern match | `case x of ...` |
| Pipe | `x \|> f \|> g` |
| Maybe | `Just x` or `Nothing` |
| Result | `Ok x` or `Err reason` |
| Sum type | `data Light = Red \| Yellow \| Green` |
| Do block | `do { x <- step1; step2 x }` |
| String interpolation | `[fmt\|hello #{name}\|]` |

That is the entire vocabulary you need for the tutorial. If any of these feel shaky, re-read the exercise — muscle memory is about repetition, not complexity.
