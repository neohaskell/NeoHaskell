## Imports you'll always have

```haskell
import Core                        -- Unqualified — Text, Int, Array, Maybe, Result, Task, etc.
import Text qualified              -- Text.trim, Text.toUpper, etc.
import Array qualified             -- Array.map, Array.filter, Array.push
import Map qualified               -- Map.get, Map.set, Map.empty
import Task qualified              -- Task.yield, Task.throw, Task.map
import Decider qualified           -- Decider.acceptNew, Decider.reject
import Json qualified              -- Json.encode, Json.decode
```

`import Core` is unqualified and gives you the essentials. Everything else is qualified by convention.

## Core types

| Type | What it is | Example |
|---|---|---|
| `Text` | A string | `"hello"` |
| `Int` | An integer | `42` |
| `Uuid` | A UUID | `Uuid.nil`, `cmd.entityId` |
| `Array a` | An ordered collection | `Array.fromLinkedList [1, 2, 3]` |
| `Map k v` | A key-value dictionary | `Map.empty`, `Map.set "a" 1 m` |
| `Maybe a` | `Just x` or `Nothing` | `Just 5`, `Nothing` |
| `Result err ok` | `Ok x` or `Err reason` | `Ok 42`, `Err "bad input"` |
| `Task err val` | Async computation that can fail | `Task.yield 42`, `Task.throw "nope"` |

## Syntax reference

### Records

```haskell
-- Define
data User = User
  { name :: Text
  , age :: Int
  }
  deriving (Show, Generic)

-- Construct
alice = User { name = "Alice", age = 30 }

-- Read
alice.name

-- Update
olderAlice = alice { age = alice.age + 1 }
```

### Pattern matching

```haskell
classify :: Int -> Text
classify n = case n of
  0 -> "zero"
  1 -> "one"
  n | n < 0 -> "negative"
    | n < 10 -> "small"
    | otherwise -> "large"
```

### Sum types

```haskell
data Event
  = UserSignedUp { userId :: Uuid, email :: Text }
  | UserDeleted { userId :: Uuid }
  deriving (Show, Generic)

describe :: Event -> Text
describe event = case event of
  UserSignedUp { email } -> [fmt|Sign-up: #{email}|]
  UserDeleted { userId } -> [fmt|Deleted: #{userId}|]
```

### Pipes

```haskell
-- Equivalent to: Text.toUpper (Text.trim someText)
someText
  |> Text.trim
  |> Text.toUpper
```

### Do blocks (Task)

```haskell
loadUser :: Uuid -> Task Text User
loadUser userId = do
  user <- fetchFromDb userId
  let greeting = [fmt|Hello, #{user.name}!|]
  Console.print greeting
  Task.yield user
```

- `<-` binds the result of a step (like `await`).
- `let` inside `do` binds a local value.
- `Task.yield` returns.
- `Task.throw` fails.

### String interpolation

```haskell
message = [fmt|User #{user.name} is #{user.age} years old|]
```

Use `[fmt|...|]` for any string you'd interpolate. Never use `++`.

## Event sourcing primitives

### Define an entity

```haskell
data Cart = Cart
  { cartId :: Uuid
  , items :: Array CartItem
  }
  deriving (Generic, Show)

instance Json.FromJSON Cart
instance Json.ToJSON Cart

type instance NameOf Cart = "Cart"

instance Entity Cart where
  initialStateImpl = Cart { cartId = Uuid.nil, items = Array.empty }
  updateImpl = update

update :: CartEvent -> Cart -> Cart
update event cart = case event of
  CartCreated { entityId } ->
    cart { cartId = entityId }
  ItemAdded { item } ->
    cart { items = Array.push item cart.items }
```

### Define events

```haskell
data CartEvent
  = CartCreated { entityId :: Uuid }
  | ItemAdded { entityId :: Uuid, item :: CartItem }
  deriving (Eq, Show, Generic)

instance Json.FromJSON CartEvent
instance Json.ToJSON CartEvent

type instance EventOf Cart = CartEvent
type instance EntityOf CartEvent = Cart

instance Event CartEvent where
  getEventEntityIdImpl event = case event of
    CartCreated { entityId } -> entityId
    ItemAdded { entityId } -> entityId
```

### Define a command

```haskell
data CreateCart = CreateCart
  deriving (Generic, Show)

instance Json.FromJSON CreateCart

getEntityId :: CreateCart -> Maybe Uuid
getEntityId _ = Nothing  -- Creation commands don't target an existing entity

decide :: CreateCart -> Maybe Cart -> RequestContext -> Decision CartEvent
decide _cmd cart _ctx = case cart of
  Just _ -> Decider.reject "Cart already exists"
  Nothing -> do
    cartId <- Decider.generateUuid
    Decider.acceptNew [CartCreated { entityId = cartId }]

type instance EntityOf CreateCart = Cart
type instance TransportsOf CreateCart = '[WebTransport]

command ''CreateCart
```

### Define a query (read model)

```haskell
data CartSummary = CartSummary
  { cartId :: Uuid
  , itemCount :: Int
  }
  deriving (Eq, Show, Generic)

instance Json.ToJSON CartSummary
instance Json.FromJSON CartSummary
instance ToSchema CartSummary

canAccess :: Maybe UserClaims -> Maybe QueryAuthError
canAccess = publicAccess

canView :: Maybe UserClaims -> CartSummary -> Maybe QueryAuthError
canView = publicView

deriveQuery ''CartSummary [''Cart]

instance QueryOf Cart CartSummary where
  queryId cart = cart.cartId
  combine cart _existing =
    Update CartSummary
      { cartId = cart.cartId
      , itemCount = Array.length cart.items
      }
```

## Decider moves

```haskell
-- Accept only if entity does not exist (creation commands)
Decider.acceptNew [events...]

-- Accept only if entity exists (update commands)
Decider.acceptExisting [events...]

-- Accept regardless
Decider.acceptAny [events...]

-- Optimistic concurrency — accept only if stream is at this position
Decider.acceptAfter position [events...]

-- Reject with a reason
Decider.reject "reason"

-- Generate a UUID inside a Decision
cartId <- Decider.generateUuid
```

## Assembling a service

```haskell
service :: Service _ _
service =
  Service.new
    |> Service.command @CreateCart
    |> Service.command @AddItem
    |> Service.command @RemoveItem
```

All commands in one service must share the same entity/event types — the compiler enforces this.

## Assembling an application

```haskell
app :: Application
app =
  Application.new
    |> Application.withConfig @MyConfig
    |> Application.withEventStore makePostgresConfig
    |> Application.withTransport WebTransport.server
    |> Application.withService cartService
    |> Application.withService orderService
    |> Application.withQuery @CartSummary
    |> Application.withQuery @OrderHistory
    |> Application.withOutbound @EmailNotifications
```

## Common errors and what they mean

| Error fragment | What it means | Fix |
|---|---|---|
| `Non-exhaustive patterns in function update` | You added an event but didn't handle it in `update` | Add a case for the new event |
| `Event type mismatch in service definition!` | Two commands in one service use different event types | Move one command to its own service, or unify the event types |
| `No instance for (Entity Foo)` | You forgot the `instance Entity Foo` declaration | Add the instance with `initialStateImpl` and `updateImpl` |
| `Couldn't match expected type 'Task Text a'` | A function returned `IO` or a raw value where `Task` was expected | Wrap with `Task.yield` or use `Task.fromIO` |
| `Variable not in scope: pure` | You used `pure` instead of `Task.yield` | Replace `pure` with `Task.yield` |

## Testing patterns

### Given-When-Then command test

```haskell
spec :: Spec Unit
spec = describe "Deposit command" do
  it "rejects when the account does not exist" \_ -> do
    let result = runDecision context (decide depositCmd Nothing noAuth)
    result |> shouldSatisfy isRejectedWith "Account does not exist"

  it "accepts a valid deposit" \_ -> do
    let account = Just (Account { accountId = someId, balance = 100 })
    let result = runDecision context (decide depositCmd account noAuth)
    result |> shouldSatisfy (producesEvent (MoneyDeposited { amount = 50 }))
```

### Query projection test

```haskell
it "counts items in a cart" \_ -> do
  let cart = Cart { cartId = someId, items = Array.fromLinkedList [item1, item2] }
  let result = combine cart Nothing
  result |> shouldBe (Update CartSummary { cartId = someId, itemCount = 2 })
```

## Forbidden patterns

Avoid these — they are not NeoHaskell style and will fight the tooling:

- `let ... in` — use `let` inside a `do` block instead
- `where` clauses — use `let` inside `do` instead
- `pure` / `return` — use `Task.yield`
- `IO a` — use `Task err val`
- `Either a b` — use `Result err ok`
- `++` for strings — use `[fmt|...|]`
- Point-free style — write the arguments
- Pattern matching in function definitions — use `case`
- Template Haskell except the provided helpers
