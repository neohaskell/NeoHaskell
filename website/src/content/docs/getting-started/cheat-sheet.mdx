## Working with Results

{/*
  Result is how NeoHaskell handles operations that can fail.
  Corresponds to Haskell's Either but with clearer names.
*/}

| I want to... | Write this |
|---|---|
| Return success | `Result.ok value` |
| Return an error | `Result.err "insufficient funds"` |
| Handle success/failure | `case result of` / `Ok value -> ...` / `Err msg -> ...` |

|---|---|

## Common Operators

| Symbol | Meaning | Example |
|--------|---------|---------|
| `\|>` | Pipe (pass left to right) | `x \|> f \|> g` |
| `::` | Type annotation | `x :: Int` |
| `<-` | Bind in do block | `result <- action` |
| `->` | Function arrow / case branch | `Int -> Text` / `True -> "yes"` |
| `_` | Wildcard (ignore this value) | `_ -> defaultValue` |

## Common Error Messages

{/*
  The 3-5 most common compiler errors that tutorial readers will encounter.
  These should be real error messages with explanations.
*/}

| Error | What it means | How to fix it |
|-------|---------------|--------------|
| `Type mismatch: expected Text, got Int` | You're using the wrong type somewhere | Check the type annotation and make sure the value matches |
| `Unknown function: map` | You forgot the module prefix | Use `Array.map` instead of bare `map` |
| `Pattern not exhaustive` | Your `case..of` doesn't handle all possibilities | Add a `_ -> ...` branch to catch the rest |
| `Unexpected token: $` | You used `$` instead of `\|>` | Replace `$` with `\|>` for the pipe operator |
| `Undefined variable: x` | You used a variable that doesn't exist | Check the spelling and make sure it's in scope |

## NeoHaskell vs. Other Languages

{/*
  Quick Rosetta Stone for common operations.
  Addresses the "I know X, how do I say it in NeoHaskell?" question.
*/}

| In JavaScript / Python | In NeoHaskell |
|---|---|
| `if (x) { ... } else { ... }` | `case x of True -> ... ; False -> ...` |
| `array.map(f)` | `array \|> Array.map f` |
| `array.filter(f)` | `array \|> Array.filter f` |
| `try { ... } catch (e) { ... }` | `case result of Ok v -> ... ; Err e -> ...` |
| `console.log(msg)` | `Console.log msg` |
| `const x = value` | `x = value` (everything is immutable by default) |
