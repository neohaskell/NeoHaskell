- Show a small, complete NeoHaskell function (5-8 lines) with annotated arrows

- Annotations point to:
  - The **type annotation** (`::`) — "this line tells you what goes in and what comes out"
    - Read it like: `functionName :: Input -> Output` means "takes an Input, gives you an Output"
    - "You can ignore type annotations when reading. They're for the compiler. Your eyes should go to the code below."
  - The function body — "this is the actual code, the type annotation above is the contract"
- Key message: "NeoHaskell reads top-to-bottom, left-to-right. No surprises."

## Reading the Pipe Operator

{/*
  The |> pipe is the single most common NeoHaskell idiom.
  Show it as "data flows left to right, like reading English."
  Compare briefly to method chaining in JS/Python for familiarity.
*/}

- Show the **pipe operator (`|>`)** in a 3-4 line chain
- Annotate: "data flows left to right — read it like a recipe"
  - Step 1: start with this value
  - Step 2: |> pass it to this function
  - Step 3: |> then pass the result to this function
- Comparison callout: "If you've seen `array.filter(...).map(...).reduce(...)` in JavaScript — the pipe operator is the same idea, just with a different shape"
- "Every time you see `|>`, just think 'then'"

## Reading a Do Block

{/*
  do blocks are how NeoHaskell sequences steps.
  Show a simple do block that does 2-3 things in order.
*/}

- Show a **`do` block** that performs 2-3 steps
- Annotate: "`do` means 'do these things in order'"
  - `name <- getInput` means "get the input and call it `name`"
  - The `<-` arrow means "get the result and give it this name"
  - Lines without `<-` are "just do this, I don't need the result"
- Key message: "A do block is a recipe. Each line is a step. They run top to bottom."

## Reading Pattern Matching

{/*
  case..of is how NeoHaskell makes decisions.
  Show a case expression that handles 2-3 cases.
*/}

- Show a **`case..of`** expression that handles 2-3 patterns
- Annotate: "`case..of` means 'look at this value and decide what to do'"
  - Each branch is a possibility
  - The `->` arrow means "if it matches this pattern, do that"
  - The compiler ensures you handle every case — nothing falls through
- Comparison callout: "Like a `switch` statement, but the compiler checks you covered every case"
