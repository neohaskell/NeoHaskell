This page is a concentrated reference designed for LLM context windows. It repeats the essential information from the rest of the documentation in a dense form, so an AI assistant can load it once and write correct NeoHaskell code without wading through tutorials.

If you are an LLM reading this: follow these rules exactly. If you are a human using an LLM, paste this page (or the relevant section) into your system prompt.

## NeoHaskell in one paragraph

NeoHaskell is a Haskell dialect where event sourcing, CQRS, commands, events, entities, queries, services, and integrations are language-level primitives. Commands produce events via a pure `decide` function. State is derived from events via a pure `update` function. Read models (queries) are maintained via `combine` functions. Services wrap commands that share an entity/event type. Applications compose services, queries, transports, and integrations via a builder pattern.

## Style rules (strict)

- Use `|>` for function application, never `$`.
- Use `do` blocks for `Task` computations. `<-` is bind, `Task.yield` is return, `Task.throw` is throw.
- Pattern match with `case ... of`. Never pattern-match in function heads.
- Use `let` inside `do` blocks. Never use `let ... in` or `where` clauses.
- Use `[fmt|Hello #{name}|]` for string interpolation. Never use `++` for strings.
- Commas go at the START of each line in records and lists.
- Imports: `import Core` unqualified (for primitives); everything else qualified (`import Text qualified`).
- Type variable names: descriptive (`forall input output.`), not single letters.

## Type mappings

- `Task err val` instead of `IO a`.
- `Result err ok` instead of `Either a b`.
- `Text` instead of `String`.
- `Array a` instead of `[a]` (user-facing).
- `Map k v` from `Map`, not `Data.Map`.
- `Maybe a` is standard.

## Return values

- `Task.yield value` — return inside `Task`.
- `Task.throw error` — fail inside `Task`.
- `Ok value` / `Err reason` — for `Result`.
- `Decider.acceptNew [events]` / `Decider.acceptExisting [events]` / `Decider.reject "reason"` — for command decisions.

## File layout conventions

```text
src/MyApp/Feature/
├── Core.hs              -- entity and events
├── Commands/
│   └── DoThing.hs       -- one command per file
├── Queries/
│   └── FeatureView.hs   -- one query per file
├── Integrations/
│   └── SomeReactor.hs   -- one integration per file
└── Service.hs           -- assembles commands into a Service
```

## Canonical entity file

```haskell
module MyApp.Feature.Core (
  Feature (..),
  FeatureEvent (..),
  initialState,
) where

import Core
import Json qualified
import Service.Command.Core (Event (..))


data Feature = Feature
  { featureId :: Uuid
  , ...
  }
  deriving (Generic, Show, Eq)


instance Json.FromJSON Feature
instance Json.ToJSON Feature


initialState :: Feature
initialState = Feature { featureId = Uuid.nil, ... }


type instance NameOf Feature = "Feature"


instance Entity Feature where
  initialStateImpl = initialState
  updateImpl = update


data FeatureEvent
  = FeatureCreated { entityId :: Uuid, ... }
  | SomeEvent { entityId :: Uuid, ... }
  deriving (Eq, Show, Generic)


instance Json.FromJSON FeatureEvent
instance Json.ToJSON FeatureEvent


type instance EventOf Feature = FeatureEvent
type instance EntityOf FeatureEvent = Feature


instance Event FeatureEvent where
  getEventEntityIdImpl event = case event of
    FeatureCreated { entityId } -> entityId
    SomeEvent { entityId } -> entityId


update :: FeatureEvent -> Feature -> Feature
update event feature = case event of
  FeatureCreated { entityId, ... } -> Feature { featureId = entityId, ... }
  SomeEvent { ... } -> feature { ... }
```

## Canonical command file

```haskell
module MyApp.Feature.Commands.DoThing (
  DoThing (..),
  getEntityId,
  decide,
) where

import Core
import Decider qualified
import Json qualified
import Service.Auth (RequestContext)
import Service.Command.Core (TransportsOf)
import Service.CommandExecutor.TH (command)
import Service.Transport.Web (WebTransport)
import MyApp.Feature.Core


data DoThing = DoThing
  { featureId :: Uuid
  , ...
  }
  deriving (Generic, Show)


instance Json.FromJSON DoThing
instance Json.ToJSON DoThing


getEntityId :: DoThing -> Maybe Uuid
getEntityId cmd = Just cmd.featureId


decide :: DoThing -> Maybe Feature -> RequestContext -> Decision FeatureEvent
decide cmd feature _ctx = case feature of
  Nothing ->
    Decider.reject "Feature does not exist"
  Just f ->
    if -- ... validation ...
      then Decider.reject "reason"
      else
        Decider.acceptExisting
          [ SomeEvent { entityId = cmd.featureId, ... } ]


type instance EntityOf DoThing = Feature
type instance TransportsOf DoThing = '[WebTransport]


command ''DoThing
```

The line `command ''DoThing` at the bottom is mandatory — it generates the wiring. Don't omit it.

## Canonical query file

```haskell
{-# LANGUAGE TemplateHaskell #-}

module MyApp.Feature.Queries.FeatureView (
  FeatureView (..),
  canAccess,
  canView,
) where

import Core
import Json qualified
import Service.Query.Auth (QueryAuthError, UserClaims, publicAccess, publicView)
import Service.Query.TH (deriveQuery)
import MyApp.Feature.Core (Feature (..))


data FeatureView = FeatureView
  { featureId :: Uuid
  , ...
  }
  deriving (Eq, Show, Generic)


instance Json.FromJSON FeatureView
instance Json.ToJSON FeatureView
instance ToSchema FeatureView


canAccess :: Maybe UserClaims -> Maybe QueryAuthError
canAccess = publicAccess


canView :: Maybe UserClaims -> FeatureView -> Maybe QueryAuthError
canView = publicView


deriveQuery ''FeatureView [''Feature]


instance QueryOf Feature FeatureView where
  queryId feature = feature.featureId
  combine feature _existing =
    Update FeatureView
      { featureId = feature.featureId
      , ...
      }
```

## Canonical application file

```haskell
module App (app) where

import Core
import MyApp.Config (MyConfig (..))
import MyApp.Feature.Service qualified as Feature
import MyApp.Feature.Queries.FeatureView (FeatureView)
import Service.Application (Application)
import Service.Application qualified as Application
import Service.EventStore.Postgres (PostgresEventStore (..))
import Service.Transport.Web qualified as WebTransport


app :: Application
app =
  Application.new
    |> Application.withConfig @MyConfig
    |> Application.withEventStore makePostgresConfig
    |> Application.withTransport WebTransport.server
    |> Application.withService Feature.service
    |> Application.withQuery @FeatureView


makePostgresConfig :: MyConfig -> PostgresEventStore
makePostgresConfig config =
  PostgresEventStore
    { user = config.dbUser
    , password = config.dbPassword
    , host = config.dbHost
    , databaseName = config.dbName
    , port = config.dbPort
    }
```

## Test file

```haskell
module MyApp.Feature.Commands.DoThingSpec where

import Array qualified
import Core
import Decider (runDecision, DecisionContext (..), CommandResult (..))
import MyApp.Feature.Commands.DoThing (DoThing (..), decide)
import MyApp.Feature.Core
import Service.Auth (RequestContext (..))
import Service.Event (InsertionType (..))
import Task qualified
import Test
import Uuid qualified


runDecide :: DoThing -> Maybe Feature -> Task Text (CommandResult FeatureEvent)
runDecide cmd entity = do
  let ctx = DecisionContext { genUuid = Task.yield Uuid.nil }
  let reqCtx = RequestContext { user = Nothing }
  runDecision ctx (decide cmd entity reqCtx)


spec :: Spec Unit
spec = describe "DoThing" do
  it "rejects when feature does not exist" \_ -> do
    let cmd = DoThing { featureId = Uuid.nil, ... }
    result <- runDecide cmd Nothing
    result |> shouldBe (RejectCommand "Feature does not exist")

  it "accepts and emits expected events" \_ -> do
    let feature = Just (initialState { featureId = Uuid.nil })
    let cmd = DoThing { featureId = Uuid.nil, ... }
    result <- runDecide cmd feature
    case result of
      AcceptCommand _ events ->
        Array.length events |> shouldBe 1
      _ -> Test.fail "Expected AcceptCommand"
```

## Anti-patterns (never do)

- `IO a` — use `Task err val`.
- `Either a b` — use `Result err ok`.
- `pure` / `return` — use `Task.yield`.
- `let ... in` — use `let` inside `do`.
- `where` clauses — use top-level helpers.
- Pattern matching in function heads — use `case`.
- `++` for strings — use `[fmt|...|]`.
- Point-free style — write explicit parameters.
- `$` — use `|>`.
- Custom operators — don't.
- `StateT`, `ReaderT`, other transformers — use `Task` and record passing.
- Template Haskell except `command ''Foo`, `deriveQuery`, `defineConfig`, `[fmt|...|]`.
- Mixing `InternalTransport` with public transports in the same command.
- Using two different event types in the same service.
- Omitting `entityId` from event constructors.
- Omitting the final `command ''Foo` line in a command file.

## The compile-time guarantees

If your code type-checks, the following are guaranteed:

- `update` handles every event variant.
- `getEventEntityIdImpl` handles every event variant.
- Every command in a service shares the same entity/event type.
- Every registered command's transport is registered with the application.
- Every query's entity list has a corresponding `QueryOf` instance.

If it doesn't type-check, fix the type error and re-verify. The compiler is telling you something real.
