## Which one should I read?

Pick the one that best matches **your current day-to-day work**. If you write Python during the day and tinker with Haskell on the weekends, read the Python page — that's the mental model you'll be carrying when you write your first NeoHaskell code.

Each page is self-contained. You don't need to read them in order.

## If your language isn't listed

These pages are aimed at the most common starting points. If you're coming from Ruby, PHP, C#, Java, Rust, Swift, or anything else, your experience will overlap significantly with the JavaScript/TypeScript or Python pages — the core observations about "how commands and events replace controllers and tables" apply across any mainstream imperative language.
