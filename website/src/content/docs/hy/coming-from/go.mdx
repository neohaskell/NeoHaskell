## Go-ում Ձեզ կարող է հանդիպել...

- Տվյալների համար structs: `type Account struct { Balance int; Owner string }`
- Վարքի համար ինտերֆեյսներ: `type Repository interface { Save(Account) error }`
- Զուգահեռաբար աշխատանքի համար goroutines: `go func() { ... }()`
- Հաղորդակցության համար ալիքներ: `ch := make(chan Event)`
- Սխալների հայտնաբերման համար հատուկ վերադարձ: `func GetAccount(id string) (Account, error)`
- `if err != nil { return nil, err }` — Go-ի սխալների հայտնաբերման ձևը
- Փաթեթներ հատուկ ներմուծումներով: `import "fmt"`, `import "github.com/user/pkg"`
- Փոխարինման ընդունիչներ փոփոխության համար: `func (a *Account) Deposit(amount int)`
- `nil` որպես զրոյական արժեք ցուցիչների, ինտերֆեյսների, զանգվածների, քարտեզների համար

## NeoHaskell-ում համարժեքն է...

- **Ալգեբրական տվյալների տիպեր փոխարեն structs**: Go structs-ը դառնում են NeoHaskell ռեկորդներ
  - `type Account struct { Balance int }` → `data Account = Account { balance :: Int }`
  - Բայց NeoHaskell-ն ունի նաև գումարային տիպեր — `data Shape = Circle Float | Rectangle Float Float`
  - Go-ն ուղղակի համարժեք չունի գումարային տիպերի (նշված միությունները պահանջում են ինտերֆեյս և տիպի փոփոխություն)
- **Հատկություններ փոխարեն ինտերֆեյսներ**: Go ինտերֆեյսները դառնում են NeoHaskell Հատկություններ (տիպի դասեր)
  - `type Repository interface { Save(Account) error }` → Հատկություն պահպանման մեթոդով
  - Գլխավոր տարբերությունը. Հատկությունները լուծվում են կոմպիլյացիայի ժամանակ, այլ ոչ թե ընթացքի ժամանակ — այդ ինտերֆեյս ուղարկման արտադրության ծախսեր չկան
  - Հատկությունները կարող են նաև արտահայտել մաթեմատիկական հատկություններ (Mappable, Appendable), որոնք Go ինտերֆեյսները չեն կարող
- **`Task` փոխարեն goroutines**: զուգահեռաբար աշխատանքի ազդեցությունները օգտագործում են `Task` տիպը
  - `go func() { result <- compute() }()` → `Task.spawn compute`
  - Խնդիրները կարող են միավորվել և տիպային ապահով են — կոմպիլյատորը գիտի, թե ինչ կարող է յուրաքանչյուր խնդիր ձախողվել
  - Ոչ մի տվյալների մրցավազք — անփոփոխ միջավայրը նշանակում է, որ ոչինչ չկա մրցավազքի համար
- **Տիպային սխալներ փոխարեն `error` ինտերֆեյս**: `Task err val` դարձնում է սխալի տիպը հայտնի
  - Go-ի `error`-ը մի ինտերֆեյս է — ցանկացած տիպ կարող է սխալ լինել, և կանչողները չգիտեն, թե ինչ սպասել
  - NeoHaskell-ի `Task NetworkError Account`-ը ասում է ճշտորեն, թե ինչ կարող է սխալ լինել
  - Ոչ `if err != nil` — դուք սխալները վերահսկում եք `case` արտահայտություններով, որոնք կոմպիլյատորը պարտադրում է
- **`Maybe` փոխարեն `nil`**: արժեքի բացակայությունը հայտնի է տիպում
  - Go-ի `nil`-ը կարող է հայտնվել ամենուր, որտեղ ցուցիչ, ինտերֆեյս կամ զանգված է օգտագործվում — լռելյայն
  - NeoHaskell-ի `Maybe val`-ը ձեզ ստիպում է բացակայության դեպքը վերահսկել կոմպիլյացիայի ժամանակ
- **Որակավորված ներմուծումներ**: `Array.map`, `Text.length` — միշտ գիտեք, թե որտեղից է գալիս ֆունկցիան
