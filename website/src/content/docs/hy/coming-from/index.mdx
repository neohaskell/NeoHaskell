Եթե ձեր նախնական փորձը ուժեղ տիպային ֆունկցիոնալ է (GHC և հարակից էկոհամակարգեր), ապա կողքի մենյուում կա [մի առանձին էջ](/coming-from/haskell/) ամբողջական համատեղելիության հղումով:

## Որ էջը պետք է կարդամ:

Ընտրեք այն էջը, որը լավագույնս համապատասխանում է **ձեր առօրյա աշխատանքին**: Եթե ցերեկը գրում եք Python, իսկ շաբաթ օրերին փորձարկում եք այլ լեզուներ, կարդացեք Python էջը՝ դա մտածելակերպն է, որ ձեզ հետ կլինի, երբ գրեք ձեր առաջին NeoHaskell կոդը:

Ամեն էջ ինքնաբավ է: Դրանք պետք չէ կարդալ հերթականությամբ:

## Եթե ձեր լեզուն ցանկում չկա

Այս էջերը ուղղված են ամենատարածված սկզբնական կետերին: Եթե ձեր փորձը Ruby, PHP, C#, Java, Rust, Swift կամ այլ բան է, ապա ձեր փորձը զգալիորեն համընկնում է JavaScript/TypeScript կամ Python էջերի հետ՝ «հրամաններն ու իրադարձությունները փոխարինում են վերահսկիչներին ու աղյուսակներին» մասին հիմնական դիտարկումները կիրառելի են ցանկացած մայնսթրիմ իմպերատիվ լեզվի համար:
