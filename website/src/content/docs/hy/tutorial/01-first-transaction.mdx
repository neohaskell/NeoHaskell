---
title: "Մաս 1. Ձեր Առաջին Տրանզակցիա"
description: "Գումար ներդնել, մնացորդ ստանալ և տեսնել, թե ինչու իրադարձությունները գերազանցում են UPDATE հրահանգներին։"
---

Ձեր նպատակն է — գումար ներդնել հաշվի մեջ և տեսնել մնացորդը։ Բայց առաջ, դուք կտեսնեք, թե ինչու ավանդական մոտեցումը (UPDATE balance SET 150) ձախողվում է, երբ հաճախորդը վիճարկում է վճարը։ Այնուհետև դուք այն կկառուցեք իրադարձությունների հետ — և կտեսնեք, որ 12 տող կոդ բավական է։

:::caution[Շուտով]
Այս էջը զարգացման տակ է։ Շուտով ստուգեք!
:::
