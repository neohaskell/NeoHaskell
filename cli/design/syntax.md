# NeoHaskell Syntax Design

**Status:** In Progress  
**Baseline:** Elm (simplicity) + Rust naming conventions  
**Last updated:** 2026-02-25

---

## Design Principles

- Elm as baseline for features (proven simplicity)
- Rust naming for familiar concepts (trait, impl)
- Domain keywords for event-sourced systems (command, event, query, entity)
- Braces instead of layout (simpler parsing)
- Hackage compatible (transpiles to standard Haskell)

### Naming Conventions (from Rust)
