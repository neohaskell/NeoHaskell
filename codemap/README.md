# codemap/ — localization assets (pipeline Phase 3, live since 2026-07-08)

The purpose: **plan-time localization in ≤3 minutes with zero exploration.**
Exact paths get resolved while planning and written into the plan; execution
never searches.

| File | Kind | Gate |
|---|---|---|
| `capabilities.yaml` | curated ontology (owns-globs, aliases, risk tags) | `check.py`: ownership exactly-once, no ghost globs, alias uniqueness |
| `extension-points.yaml` | curated — where NEW things go | `check.py`: paths + skills exist |
| `signatures/*.txt` | **generated** hoogle-format API surface | CI sync check (regenerate + diff) |
| `MAP.md` | **generated** render | regenerated by `./dev codemap` |
| `.doc-ratchet` | generated counters | `check.py`: may only decrease |
| `routing-smoke.yaml` | curated routing test set | `≥8/10` on map changes (LLM-run, scripted protocol below) |

## Routing procedure (agents: this is the contract)

1. **Existing code?** Match the request against `capabilities.yaml` — select
   capability IDs from the closed list; `aliases` bridge intent vocabulary
   ("serve a SPA" → `http-transport`). Never invent paths.
2. **New code?** Match against `extension-points.yaml` `triggers` — the row
   tells you create/register/tests locations and the driving skill.
3. **Symbol level:** grep the capability's API surface in `signatures/*.txt`
   (one Read; never open source files to discover APIs).
4. **Edit-set expansion:** `./dev who-calls <symbol> [module]` — who breaks if
   this signature changes, grouped by capability. (`./dev hiedb` builds the index.)
5. **Write exact paths + symbols into the plan.** Low confidence or empty
   match → escalate model tier; never guess.

## API discovery (Phase 4) — transcribe, never recall

Training-data APIs do not exist in this repo. Resolve symbols at **plan time**
into the plan's `uses:` list; execution transcribes them, never recalls:

- `codemap/api-hot.md` — frequency-ranked card of what this repo actually calls,
  with doctest examples (cut modules listed in the card trailer).
- `./dev api "Text -> Maybe Uuid"` — hoogle type search: the NeoHaskell surface
  ranks first; vanilla (dependency closure + boot libs) ranks below with a
  disclaimer whenever present (omitted when empty; exit 3 = vanilla-only).
  Query in dialect types (per AGENTS.md's style table) to hit the surface directly.
- `codemap/phrasebook.md` — doctest-verified usage patterns (gate: test.yml
  `doctest` job; thin coverage is the doc backlog, ratcheted via
  `undocumented_doctest_modules`).
- GHC "not in scope" in `./dev check` output = an invented API — resolve it via
  `./dev api`; pipeline runs record this per stage as `invented_api_events`.

## Routing smoke protocol (map-change PRs)

`routing-smoke.yaml` holds request → expected-capability pairs (including
vocabulary-gap probes). A fresh-context agent routes each request using ONLY
steps 1–2 above (without reading the `expected:` fields), then compares.
Pass bar: **≥8/10**. Record the score in the PR body.

## Regeneration

```sh
./dev codemap        # signatures + MAP.md + ratchet (after API changes)
./dev codemap-check  # validity gate (also in CI: checks.yml)
./dev hiedb          # symbol reference DB (local only, gitignored)
```

Governing rule applies: curated files are CI-gated, generated files carry a
generation banner and a sync check. Escape hatch for vocabulary gaps: add an
alias, not a workaround.
